## PageSpeed Insights 継続モニタリング方針

FREE AGENDA の主要なパフォーマンス変更ごとに PSI を再実行し、スコアとクリティカルパス指標を継続的にログ化するための方針です。

### 1. 計測のタイミング

- 以下のような変更を本番 or ステージングに適用する前後で必ず計測する:
  - 画像（HERO / favicon）の差し替え・圧縮率変更
  - JS バンドル削減やコード分割
  - フォント・外部スクリプト（Analytics など）の読み込み戦略変更
  - インフラ構成変更（S3/CloudFront 導入など）

### 2. 手順

1. `https://search-freeagenda.fun` を PageSpeed Insights に入力。
2. **モバイル / デスクトップ** 両方で計測。
3. 結果を `docs/pagespeed-history.md` に「エントリ 2, 3, ...」として追記する:
   - 日付
   - 変更内容（例: 「HERO 画像を WebP に差し替え」）
   - モバイル / デスクトップのパフォーマンススコア
   - LCP, FCP, INP, CLS
   - クリティカルパス最大待ち時間
   - 自社オリジンの転送サイズ・初回ドキュメントサイズ

### 3. 運用のゴール

- モバイルのパフォーマンススコアを **安定して 95 以上** に保つこと。
- クリティカルパス最大待ち時間と LCP を、機能追加を行いながらも増やしすぎないよう監視すること。
